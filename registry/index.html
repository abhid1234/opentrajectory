OpenTrajectory Registry

The open hub where every harness's trajectories meet one format.

Each harness records the same spine of an agent run in its own vocabulary. The registry tracks the adapters that normalize them into the one portable .ot.json, and publishes the canonical conformance corpus every adapter — first-party or community — is measured against. Static, zero-dependency, vendor-neutral.

Harness adapters

The harnesses that emit OpenTrajectory today. Verified = checked first-hand against a real on-disk session; provisional = built from the documented schema, not yet a validated real export.

Conformance corpus

Nine canonical documents, one per shape an adapter must get right. Make your emitter produce these and pass ot validate, and you conform. Click any case to read it.

Add your harness

The registry is community-owned. To add a harness, open a PR that contributes one canonical case and (ideally) an adapter:

  1. Capture a real run from your harness and normalize it to .ot.json (see the four adapters in packages/capture/src/from-*.ts as references).
  2. Add conformance/harness-<yours>.ot.json + a manifest.json entry with its must invariants.
  3. Validate locally, then open the PR:
# your output, checked by the same validator the CLI + GitHub Action run
node tools/ot-validate.mjs path/to/your-output.ot.json
node conformance/check.mjs        # corpus + invariants + no orphans
node registry/build.mjs           # regenerate this page's data

v2 roadmap: this static page is the seed. A hosted registry (search, side-by-side benchmark, submission flow) is the next layer — but the format, corpus, and validator that make it meaningful already ship here, today.
